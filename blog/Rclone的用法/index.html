在Docker-Compose中挂载Rclone

    rclone:
      image: rclone/rclone:latest
      container_name: rclone
      restart: always
      environment:
        - PUID=0
        - PGID=0    
      cap_add:
        - SYS_ADMIN
      security_opt:
        - apparmor:unconfined
      devices:
        - /dev/fuse
      # 一些参数
      # https://github.com/alist-org/alist/discussions/4859
      # 需确保/mnt/pan文件夹存在
      command: mount --umask=0 alist:/ /mnt/pan --vfs-cache-max-size 30G --vfs-cache-max-age 2h --vfs-cache-mode full --allow-non-empty  --multi-thread-streams 8 --header "Referer:https://pan.baidu.com/"  --header "User-Agent:pan.baidu.com" --buffer-size=256M --use-mmap -v --transfers 1 --timeout 60m
      user: 0:0
      tty: true #用于确保即便失败也可以 docker exec -it 
      volumes:
        - ./rclone/config/rclone:/config/rclone
        - ./rclone/mnt:/mnt:shared
        - /etc/passwd:/etc/passwd:ro
        - /etc/group:/etc/group:ro

关于挂载缓存相关的选项

# https://hostloc.com/thread-930972-1-1.html  
--vfs-cache-max-size 30G --vfs-cache-max-age 2h --vfs-cache-mode full

docker exec -it rclone sh

缓存持久化

rclone的缓存在~/.cache下，缓存持久化有助于重启rclone后继续上传未完成的任务
配置类似 - ./rclone-alist/cache:/root/.cache 这样的路径映射即可，若存在大量未完成的上传任务，此时重启rclone并不会立即挂载上去，需要等待一定的时间才会挂载成功

占位

若要在docker中使用rclone挂载的目录，但又不希望在目录还未成功挂载的时候被写入，可以在还未挂载的时候使用类似如下的指令mkdir /mount/point && chmod 000 /mount/point && chattr -i /mount/point这样将创建一个文件夹，该文件夹无法写入以防止程序误写入本地磁盘中，直到rclone挂载成功后会将其顶替

使用Rclone架设webdav

rclone serve webdav

    webdav-gd:
      image: rclone/rclone:latest
      container_name: webdav-gd
      networks:
        - main
      command: serve --htpasswd /auth/htpasswd --addr 0.0.0.0:80 webdav gd:/ #远程文件提供webdav
    #   command: serve --htpasswd /auth/htpasswd --addr 0.0.0.0:80 webdav /webdav #本地文件提供webdav
      volumes:
        - ./rclone/config/rclone:/config/rclone
        - /auth:/auth #具体参考htpasswd的用法
        # - ./localfiles:/webdav 
      restart: unless-stopped
