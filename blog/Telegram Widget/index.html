Telegram Widget.
Posted on 2022-04-17 :: Updated on 2022-12-05 :: Tags: Web, Telegram

   几个月之前使用Telegram Widget发现其使用免费域名tk,ml,ga时点击登录无任何反应，今天没事又试了一下发现可以使用免费域名ml登录了，于是来这记录一下在操作的过程中踩到的一些问题。

  1. Refused to frame 'https://oauth.telegram.org/'
       这个问题的原因在于在本地调试时使用了非80,443的端口号，参考
  2. 一直登录不上
       通常需要清理浏览器缓存重新点击widget在测试或者使用无痕模式，无痕模式每次启动时重新开一个页面进行测试反正缓存问题，然后要到telegram中将前面的登录给处理了否则可能影响到后续登录。
  3. 本地https调试
       原本以为第一个问题时由于本地采用的http导致的，于是找到了mkcert这个项目，可以用来本地安装证书生成密钥,之后只需要在服务端引入密钥即可。
   mkcert -install #安装本地证书  
   mkcert example.com "*.example.com" example.test localhost 127.0.0.1 ::1 #生成密钥
