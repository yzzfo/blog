  最近使用Emby的时候发现CPU占用率奇高，发现是在Exsi下N6005的核显并没有参与到硬解的工作中去，于是在网上多方搜索发现最新的PVE7.2似乎是可以LXC容器的形式来使用核显硬解，所以在这里记录一下使用PVE的过程。

安装PVE

  PVE就是一个Debian的定制版，安装过程和其他操作系统没什么区别，主要记录一下安装后的相关设置。

  1. 设置直通特性

    nano /etc/default/grub 替换GRUB_CMDLINE_LINUX_DEFAULT="quiet"

    为 GRUB_CMDLINE_LINUX_DEFAULT="quiet intel_iommu=on" amd的cpu则将inter改amd即可

    update-grub

    nano /etc/modules

    填入如下内容

    vfio
vfio_iommu_type1
vfio_pci
vfio_virqfd

    update-initramfs -u -k all

    重启PVE

  2. 安装Openwrt

    qm importdisk 102 /var/lib/vz/template/iso/openwrt.img local-lvm

    以上代码102对应的为虚拟机， /var/lib/vz/template/iso/openwrt.img对应的是上传的路径

安装LXC容器

  为了使用核显，这里选择Debian11的模板进行安装。

  1. 更新内核至5.17.5

    curl -1sLf 'https://dl.cloudsmith.io/public/pve-edge/kernel/gpg.8EC01CCF309B98E7.key' | apt-key add -
echo "deb https://dl.cloudsmith.io/public/pve-edge/kernel/deb/debian bullseye main" > /etc/apt/sources.list.d/pve-edge-kernel.list
apt update
apt install pve-kernel-5.17.5-edge -y
update-grub
reboot

  2. 查看当前内核

    uname -a
dpkg --get-selections |grep kernel

  3. 安装Debian11模板，创建时取消非特权容器的勾选

  4. 挂载核显（在PVE的shell中执行）

    ls -l /dev/dri

total 0
drwxr-xr-x 2 root root         80 May 21 16:19 by-path
crw-rw---- 1 root video  226,   0 May 22 08:54 card0
crw-rw---- 1 root render 226, 128 May 21 16:19 renderD128

nano /etc/pve/lxc/104.conf

#粘体一下内容
#这里的226和128对应ls下显示的东西
lxc.cgroup2.devices.allow: c 226:0 rwm
lxc.cgroup2.devices.allow: c 226:128 rwm
lxc.cgroup2.devices.allow: c 29:0 rwm
lxc.mount.entry: /dev/dri dev/dri none bind,optional,create=dir
lxc.mount.entry: /dev/fb0 dev/fb0 none bind,optional,create=file
#为了在LXC可以使用Docker需要加入如下配置
#否则会出现 AppArmor enabled on system but the docker-default profile could not be loaded
lxc.apparmor.profile: unconfined

#强制开启硬解
nano /etc/modprobe.d/i915.conf

#添加
options i915 enable_guc=2
undate-initramfs -u -k all

  5. 在Debina11中安装nyanmisaka/jellyfin:latest的docker容器，该容器携带了相关核显驱动，emby经过尝试发现还无法正常驱动核显，挂载docker时要进行设备挂载，在docker-compose中如下配置

      jellyfin:
    image: nyanmisaka/jellyfin
    container_name: jellyfin
    environment:
      - PUID=0
      - PGID=0
      - TZ=Asia/Shanghai
    volumes:
      - ./jellyfin/library:/config
      - /mnt/nas:/data/root
    ports:
      - 8096:8096
      - 8920:8920 #optional
      - 7359:7359/udp #optional
      # - 1900:1900/udp #optional
    devices:
      - /dev/dri:/dev/dri # VAAPI/NVDEC/NVENC render nodes
    restart: unless-stopped

  6. samba挂载

    在LXC中挂载Samba和通常系统下挂载有些许不同，在LXC中非特权容器需要在高级选项中勾选samba才可以使用，但无论是特权还是非特权均无法让其开机自动挂载抑或是使用x-systemd.automount这个选项来进行按需挂载，解决方法是在PVE上进行samba挂载，之后映射到LXC中，以LXC容器101为例

    nano /etc/pve/lxc/101.conf

#添加如下内容
mp0: /mnt/宿主机名称/NFS挂载目录/,mp=/mnt/LXC容器中希望的挂载目录

黑群晖

由于网上黑群晖安装方式感觉十分多样，于是先记录一下自己在PVE中安装黑群晖时的方法

  1. 下载引导 黑群DSM7.X引导文件（2022年6月3日更新） - GXNAS博客

  2. 创建虚拟机，使用qm importdisk将引导载入虚拟机，并选择SATA模式

  3. 挂载另一款硬盘至虚拟机上用作存储

  4. 开机后查看局域网内的群晖IP进入web后台

  5. 根据web的安装引导下载对应的系统

  6. 断开网络之后在进行系统安装
