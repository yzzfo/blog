在Openwrt中有许多关于DNS的设置，为了理清各种关系和流程，故在此记录一下。

基础DNS设置

  1. Wan口DNS设置: 路由器访问网络时使用的DNS地址
  2. Lan口DNS设置: DHCP下发给其他设备的DNS地址
  3. DHCP/DNS该功能其实是dnsmasq，其uci对应/etc/config/dhcp.conf
    3.1 DNS 转发: 将请求本地53端口的请求转发给设置的DNS
    3.2 解析文件: 通常该文件是/tmp/resolv.conf.d/resolv.conf.auto,该文件中的DNS地址为Wan口中的DNS地址。
    3.3 两个选择之间的关系: 经过测试，只要其中一个选择设置正确即可通过使用nslookup获取到解析的域名，故推测两者为并行关系。

总结

  对于设置静态DNS的人来说，这几个选项对其没有任何影响，只有当Lan口DNS设置为当前路由IP或者不填时（默认路由IP），其他机器才会访问dnsmasq进而产生影响。

DNS防污染

  由于通常情况下DNS是明文传输很容易做到篡改，也就造成了国内去访问国外域名时很容易获取到受到污染的IP地址，这种情况下就两种选择，一种是直接本地解析域名即host文件，另一种走加密的DNS，本地解析这种方法过于粗犷，所以还是采用加密DNS，但为了DNS能够有更好的速度应该采用特定域名查询采用加密，正常的域名走老方法。

gfwlist

  gfwlist是一个包含了大量国内被墙或污染的域名列表，通过它我们可以获取什么域名应该走加密的DNS请求。

SmartDNS

  SmartDNS是一个本地DNS服务器，可以设置多个上游DNS服务器，它自带两个DNS服务器设置，这正是我们想要的，一个用来走加密查询，一个用来走常规查询。

SSR Plus

  是一款代理软件，其支持使用gfwlist进行分流操作，在运行模式选择GFW运行模式以及DNS解析方式中选择使用本机端口为5335的DNS服务，即可当收到包含在gfwlist中的域名需要进行DNS查询时将会走到SmartDNS的5335服务中去。

SSR Plus分流的原理

  SSR Plus分流是基于dnsmasq实现的，具体可查看/var/dnsmasq.d/dnsmasq-ssrplus.d下的文件，在该文件夹下面包含了许多生成的dnsmasq的分流规则

server=/google.to/127.0.0.1#5335    // 域名google.to通过127.0.0.1#5335 去解析
ipset=/google.to/gfwlist    // 域名google.to解析后的IP设置到gfwlist集合中

可以看到这些规则不但定义了分流规则，也为之后的代理对ip进行了分类，后续通过iptable将所有处于gfwlist集合中的ip转发给代理。(ipset list 可以看到所有集合)

DNS查询代理

  分流过后DNS污染其实是已经解决了，但返回的IP其实是离我们当前路由器最近的节点，这是由于我们查询时是使用我们本机的IP查询的，为了让代理服务器访问最适合他的IP，我们应该让代理服务器去查询DNS，这时候我们应该让DNS的查询请求也走代理才可以。
DNS走代理有几个注意点:

  1. 例如 https://dns.google/dns-query这个https的DNS查询服务，如若加密分流都是这种域名类型的，要使用这个查询服务就必须先解析dns.google这个域名，所以我们不能让dns.google这个域名的查询走当前设置的加密分流，因为这将陷入死循环，为了解析dns.google而去通过dns.gooogle解析dns.gooogle。只有当存在明确IP地址的dns存在时才能让其走加密，否则将陷入循环之中。
  2. 尽量直接使用IP地址类型的DNS，这样就可以直接在代理中强制设置IP走代理，也无需注意第一点中的问题了。
  3. 代理服务器的域名解析不可走代理。

dnsmasq设置

  在SSR Plus仅仅设置了处于gfwlist内时的行为，需要在dnsmasq中修改默认行为来适配gfwlist外的域名，dnsmasq中将DNS 转发设置为127.0.0.1#6335同时关闭解析文件即可。

在 OpenClash 中设置 AdGuard

DNS 查询顺序 Client => OP: 53 => AD => OpenClash

之所以 AD 要位于 OpenClash 之前是因为 OpenClash 在查询需要代理的域名时将采用远程解析，即跳过上游 DNS 直接转发至代理服务器去解析 DNS。

所以设置方法为，AD 中设置劫持 53 端口，将上游 DNS 设置为 OpenClash 即 127.0.0.1:7874 在 OpenClash 中关闭 DNS 劫持，并设置上游 DNS。
