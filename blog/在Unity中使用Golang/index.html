在Unity中使用Golang

最近用Golang写了一些工具，于是就想着能不能移植到Unity中使用，众所周知Golang可以很方便的跨平台编译也可以和C交互，而Unity也是可以和C交互的，都可以通过动态链接库的形式进行连接，于是乎开始尝试在Unity中使用Golang的库。

在Golang中编译链接库是使用CGO进行编译的，这就导致不能像编译普通Golang程序一样轻松跨平台，需要解决如下几点:

  1. 安装指定平台的GCC和G++
  2. 启用CGO
  3. 设置目标架构和系统
  4. 设置使用的GCC和G++

安装GCC和G++

  • 对于Win来说有3种途径: Mingw，MSys2，tdm-gcc，前两种由于网络原因并不怎么好安装，即便安装了也还存在一些坑需要配置，个人感觉对于Golang来说tdm-gcc是最好的选择。
  • Linux, 我这边使用的Ubuntu系统，其具体安装方式就是直接apt

  • Android虽然也是Linux系统但他不能直接用Linux的库，需要使用NDK进行编译才能使用，在Win下安装NDK，采用NDK中的编译器去编译,安装NDK后设置环境变量NDKROOT和%NDKROOT%\toolchains\llvm\prebuilt\windows-x86_64\bin
    NDK中的编译器
armv7a-linux-androideabi29-clang.cmd
armv7a-linux-androideabi29-clang++.cmd

其中数字29代表Android10，也和Unity中PlayerSetting的安卓API对应，具体可以百度关键字 安卓版本与API

设置Golang

以安卓为例，设置armv7架构的安卓

export GOARCH=arm
export GOOS=android
export CGO_ENABLED=1
export CC=armv7a-linux-androideabi29-clang.cmd
export CXX=armv7a-linux-androideabi29-clang++.cmd

Compile

接下来就是编译了
go build -buildmode=c-shared -o libv2free.so v2free.go
编译导出的名字很重要，libxxx.so xxx为库名,如果直接导出xxx.so会报找不到Dll的错误，原因看一篇博文是说Android中查找库是按照这种命名规范来找的。

接下来就是Unity的调用了

将so或dll，拖入unity中后设置好他们正确的属性，比如运行的平台，架构等等，这样Unity才可以正确的编译，在Editor中，运行调用的是本地架构的dll，如果只有安卓没有本地架构的dll那就只能安卓测试了。对于安卓来说，设置好项目里的Android API与导出的对应，或者说导出的时候设置的与项目一致，否则可能Crash。
调用方式如Golang编译一文中所述,接上后就能正常使用了。

踩到的坑

感觉像是Unity缓存或者设置的问题，在老项目中编译的安卓会各种闪退，关闭增量GC又好一些，继续关闭多线程渲染，反正各种方法试过还是会出现概率闪退问题，logcat查了调用堆栈指向了GC Crash属实看不懂了，重新开了个Unity导出就好了，感觉以后遇到奇怪问题还是重开个项目试试比较好。

调试

安卓ADB调试，在2019中可以下载Logcat插件了，对于安卓ADB调试十分方便，对于ADB的Crash堆栈可以通过IDA和Il2CppDumper两个东西对应着查看，但也可以用更方便的方式，导出在Build时勾选symbols.zip以及各种Debug的东西，这样配合Logcat插件就很舒服了。
