之前使用WordPress来构建自己的博客系统，奈何自己的VPS性能太孱弱了(1C1G)而且还要在Docker中运行使得要打开WP部署的网页都得几分钟，所以改用Hexo来构建自己的博客了， Hexo和WP的区别就在于，Hexo是静态博客，WP是动态的，静态的好处当然就是只需要服务器直接发送文件给浏览器就行了，不需要动态去生成网页然后在发送给浏览器，这种只提供文件传输，其他东西都交给浏览器去干的方案正是我这台VPS所需要的。

要使用Hexo首先应该大体了解一下他的工作流程，Hexo可以看作就是一个markdown转html的软件，但不同于Typora这种markdown软件的导出为html,Hexo导出时会进行一些额外的操作，比如构建文章的数据库等等的额外操作，并将这些数据和js进行关联，最终组成整个博客。也就是说Hexo导出的东西就两类，js构成的用户操作和数据。

使用步骤

  1. 安装Node.js
  2. npm install -g hexo-cli
  3. hexo init [folder] 初始化hexo项目
  4. hexo new <title> 创建一篇markdown文章
  5. 调试和部署
    • hexo server 开启hexo服务,此时可在localhost:4000看到博客效果
    • hexo clean && hexo g -d hexo项目构建，将构建的结果即public文件夹上传到服务器即可完成部署
