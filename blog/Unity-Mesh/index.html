SharedMesh和Mesh的区别

sharedMesh是指MeshFilter组件中共享的mesh，它会影响所有使用这个mesh的对象。如果你想修改一个对象的mesh而不影响其他对象，你可以使用MeshFilter组件中的mesh属性，它会返回一个新的mesh副本。

Submesh和Materials

Submesh 是指一个 mesh 中的一部分，可以用来给不同的部分应用不同的材质。对应到 Blender 中则是一个 object 中存在多个分开的网格，如下图 mesh 中存在两个 submesh(材质不同的时候两个，相同的话一个)。

Submesh和Materials对应的关系为

  • 若Submesh数量等于Material数量，则index对应赋值
  • 若Submesh小于Material数量，相等部分对应赋值，多出的Materials重复作用于最后的Submesh上
  • 若Submesh大于Material，相等部分对应赋值，剩下的Submesh都使用最后的一个Material

CombineMesh

CombineMesh有一个参数叫mergeSubMeshes，它可以控制合并后的mesh是否只有一个submesh。如果你设置为true，所有的submesh都会合并成一个submesh。如果你设置为false，每个submesh都会保留在不同的submesh中即需要一个drawcall。

CombineMesh的好处
当mergeSubMeshes为false的时候，可以看出来他的drawcall并没有降低，对于类似换装系统这样的功能来说倒是可以将多个SkinmeshRender变成一个SkinmeshRender，可能从结果上来看可以减少一些渲染状态切换之类的东西，例如每次渲染Unity都需要设置一些渲染参数，比如材质、光照、混合模式等。如果你把多个mesh合并成一个mesh，你就可以减少这些参数的设置次数，从而提高效率。

Triangle

在存储顶点以及确定它们的三角面时通常会有如下几种方法:

  • 三角形条带法：这是一种绘制三维图形的方法，它可以减少顶点数据的冗余，提高渲染效率。它的原理是，按照一定的顺序指定一系列顶点，然后每相邻的三个顶点构成一个三角形。例如，如果指定了7个顶点v1,v2,v3,v4,v5,v6,v7，则会绘制出4个三角形：v1v2v3, v2v3v4, v3v4v5, v4v5v6。

  • 三角形扇法：这也是一种绘制三维图形的方法，它可以用较少的顶点数据来表示圆锥或球体等曲面。它的原理是，以一个顶点为中心，与其他顶点依次构成一系列扇形的三角形。例如，如果指定了6个顶点v1,v2,v3,v4,v5,v6，则会绘制出5个三角形：v1v2v3, v1v3v4, v1v4v5, v1v5v6, v1v6v2。

  • 四边形条带法：这是一种绘制四边形网格的方法，它可以用两条线把一系列四边形交错起来。它的原理是，按照一定的顺序指定一系列顶点，然后每相邻的四个顶点构成一个四边形。例如，如果指定了8个顶点 v1,v2,v3,v4,v5,v6,v7,v8，则会绘制出3个四边形：

  • v1-v2-v4-v3

  • v3-v4-v6-v5

  • v5-v6-v8-v7

  • 索引缓冲法：这是一种优化渲染性能和内存使用的方法，它可以避免重复存储相同的顶点数据。它的原理是，使用一个数组来存储所有不重复的顶点数据（称为顶点缓冲），然后使用另一个数组来存储每个图元（如三角形或四边形）所需的顶点索引（称为索引缓冲）。例如，如果要绘制一个立方体，则只需要8个不同的顶点和36个索引。

在Unity在Meshfileter的triangles属性就是使用的索引缓冲法。对于不同的存储方法，在OpenGL中也是有对应不同的绘制方法，有些图元类型是每三个点认为一个三角的索引缓冲法，比如GL_TRIANGLES，有些图元类型是用的三角形条带法，比如GL_TRIANGLE_STRIP。

三角形条带法的一些劣势

人物的网格通常是非常复杂和不规则的，很难用一系列连接的三角形来表示。如果要用三角形条带法来绘制人物，就需要分割成很多小的条带，并且使用很多退化的三角形来连接它们，这样就会降低效率和质量。

三角形条带法有可能无法画特点拓扑的模型，它只能用一系列连接的三角形来表示网格，而不能自由地指定任意顶点之间的连接关系。 这样就会导致一些问题，例如：

  • 三角形条带法不能绘制具有洞或凹陷的模型，因为这些模型需要断开或跳过一些顶点。
  • 三角形条带法不能绘制具有复杂曲面或细节的模型，因为这些模型需要更多或更少的顶点来控制曲率或密度。
    • 如果模型需要在某些地方有很高的法曲率，就需要增加顶点来使三角形条带能够适应这种弯曲变化。
    • 如果模型需要在某些地方有很低的法曲率，就需要减少顶点来使三角形条带能够保持平滑和均匀。
    • 但是三角形条带法只能用顺序的顶点来表示网格，不能根据不同位置和方向调整顶点数量和位置。
  • 三角形条带法不能绘制具有动画或变形的模型，因为这些模型需要更好的拓扑结构来保持平滑和连贯。
  • 三角形条带法的定义是：一系列连接的三角形，其中每个新顶点与前两个顶点共享一条边。三角形条带法的实现是：使用一个数组或缓冲区来存储顶点数据，然后按照固定的规则来遍历这些顶点并绘制三角形。因此，如果要改变顶点数量或位置，就需要重新创建或修改数组或缓冲区，并重新遍历和绘制所有顶点和三角形。这样就会增加计算和内存开销，并降低渲染效率。

索引缓冲法则可以更灵活地指定任意顶点之间的连接关系，从而可以绘制任何形状的网格，包括人物。
