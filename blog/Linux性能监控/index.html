今天在弄Drone的时候发现发现性能十分不稳定，同一个任务运行时间2-10分钟不等，差距实在是太大了，于是想看看问题出在哪里，找了一圈查看性能的工具所以在这记录一下。

命令行工具

  1. top 自带工具，用来看看CPU和MEM
  2. iotop 用来查看当前IO占用的工具
  3. free 查看内存统计信息
  4. 查看进程swap使用情况 smem -k -s swap

Netdata

命令行工具通常也就是拿来看看当前情况如何，有时候想看看详细的数据统计或图表，这时候可以安装Netdata来完成该需求，Netdata是一个开源的方案，而且提供了前端界面用起来十分的方便，统计的数据也很详细。

通过Docker Compose安装

version: '3'
services:
  netdata:
    image: netdata/netdata
    container_name: netdata
    hostname: example.com # set to fqdn of host
    ports:
      - 19999:19999
    restart: unless-stopped
    cap_add:
      - SYS_PTRACE
    security_opt:
      - apparmor:unconfined
    volumes:
      - netdataconfig:/etc/netdata
      - netdatalib:/var/lib/netdata
      - netdatacache:/var/cache/netdata
      - /etc/passwd:/host/etc/passwd:ro
      - /etc/group:/host/etc/group:ro
      - /proc:/host/proc:ro
      - /sys:/host/sys:ro
      - /etc/os-release:/host/etc/os-release:ro

volumes:
  netdataconfig:
  netdatalib:
  netdatacache:

浏览

浏览的方案有两种:

1. 使用诸如Nginx或者Caddy之类的服务端进行反向代理以此来使用Netdata

2. 在Docker部署时链接Netdata官方提供的服务，通过官网来查看详细数据

  1. 登录 https://app.netdata.cloud
  2. 注册并登录后找到Nodes相关选项
  3. 选中Nodes选项后找到Docker相关的选项，复制里面提供的Docker代码在本机运行即可。
  4. 安装完成之后在该网站就能看到服务器的性能数据了。

IOSP

IOSP意为磁盘每秒的读写次数，有些VPS厂家是会锁这个值，之前购买VPS的时候没怎么关注过这个性能指标，但最近使用Cloudcone的时候发现指令执行起来出奇的慢，最后使用https://github.com/masonr/yet-another-bench-script这个脚本跑了一下发现在4k block的情况下IOSP这个指标相较另一台服务器来说有巨大的差异(Cloudcone 100+,Servarica 15k+),于是出现一个很有意思的事情发生Servarica测出来的读写速度没Cloudcone快，但实际体验来看Servarica明显好很多，执行各种任务都没有明显延迟，而Cloudcone则是每次等几秒甚至数分钟才能有反应。在执行指令时Cloudcone的iowait会达到100%，而Servarica则基本为0。这就有点像是拿货轮送外卖的形象了，虽然读写大，但是每次间隔太长了。
