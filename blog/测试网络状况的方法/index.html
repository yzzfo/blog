测试网络状况的方法.
Posted on 2023-12-10 :: Tags: Tools

在使用代理的时候会需要对他们进行一些测试来看看他们的效果如何，通常诸如ping或tcping之类的手段也只能去简单测试他们的连接速度，在使用tun模式的时候大部分情况下也只能使用软件自带的tcping进行简单测速（类似nekoray不支持hysteria2进行tcping测试），ping则无法通过tun进行转发从而导致无法使用。 当我们想测试一下当前客户端到服务器的网络波动以及速度的时候上面的方法自然无法满足，这个时候我们可以通过iperf3来进行测试，通过在服务器运行iperf3，客户端代理本地iperf3则可以用于测试客户端到服务器直接的网络质量。

#服务器
iperf3 -s

#开启代理后运行
#客户端
# udp 测试
iperf3 -u -t 30 -c server_ip

# tcp 测试
iperf3 -t 30 -c server_ip

通过这种测试方法也可以推测出代理针对不同流量所使用的策略，例如vless协议中代理udp是走的udp over tcp所以在iperf3的测试中udp丢包为0，但客户端测试使用hysteria2转发udp的时候出现了丢包，而当服务器也走该代理且运行客户端同样的指令时却没出现丢包，通过这两点推测hysteria2通过修改quic协议在处理udp转发的包的时候使用的不可靠传输模式即不对udp的转发包进行重传之类的，类似于udp直连，这也导致出现了丢包的情况。和shadow sock中对待udp类似。
