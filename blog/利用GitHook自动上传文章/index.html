利用GitHook自动上传文章

涉及的工具

- Git
- Pandoc
- WordPress
- Golang

流程

在markdown工具中写好文章后，保存到git中，触发githook后运行pandoc导出html形式的文章，最后通过golang调用wordpress的api进行文章创建或更新操作

git服务器的搭建

我这边服务器选择用本地的树莓派来搭建，git服务端和客户端的操作流程即便一样，网上大多教程采用git init --bare来初始化服务端的仓库，但这样操作出来的仓库似乎就只有存储的作用，故我这边还是选择git init直接创建一个仓库,至于要不要创建一个专门的git用户用来进行仓库通信就全凭喜好了。
仓库创建好后自然是客户端链接上去试试，链接过程和链接Github的仓库一样，git remote add 服务名 ssh://username@ip:port，至此仓库是已经通了。

安装Pandoc

在树莓派中安装pandoc,pandoc已经在apt中包含了，直接sudo apt-get install pandoc -y，关于pandoc的使用Demo

文章上传

接下来考虑如何上传文章,这里调用golang中的一个wordpress库，在网上看到wordpress也有xml-rpc试了一试感觉还是调库来的舒服，主要容易遇到wordpress拒绝访问的问题，解决方法就是用户->所有用户->编辑用户->添加新的应用程序密码，用这个密码代替登陆密码即可访问。关于这个库的使用见restful api

GitHook

剩下就是GitHook了,hook事件选择post-receive,该事件的时间点在push结束,可以在该事件中获取到前后的commit的信息，可以用来筛选该操作的文件,具体代码如下。

#!/usr/bin/sh
#读取git hooks的输入
read old_value new_value ref_name
echo "post-receive"
echo ${new_value}
echo ${old_value}
echo ${ref_name}

files=`git diff --name-only ${old_value} ${new_value}`

unset GIT_DIR
cd ..
git checkout ${new_value}

#TODO Sth

mkdir "export"
git checkout main
#files=`ls`
echo "这些文件发生了变化--> "$files
echo "\033[32;40m 开始导出html \033[0m"
for file in $files
do
  file=`echo $file|sed 's/\"//g'`   #去除引号
  if [ "${file##*.}"x = "md"x ];then
    pandoc --toc --standalone --mathjax -f markdown $file -o ./export/${file%.*}.html
    /home/pi/word_arm ./export/${file%.*}.html
  fi
done


git checkout --detach
echo "post-receive end"
