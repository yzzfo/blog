UnityEditor

大体流程

指令形

类似win上的刷新按钮，点击即执行一个固定的功能

//最基本的使用方式
//只需静态方法即可，无需其他要求
[UnityEditor.MenuItem("Tools/xxx")]
public static void FuncA()
{
    //Todo sth
}

//带状态的按钮
//比如: Unity在编译中，则该禁用该按钮
//第二个参数，指明了该方法不是用来判断状态的函数，而是执行函数
[UnityEditor.MenuItem("Tools/xxx",false,0)]
public static void FuncA()
{
    //Todo sth
}

//第二个参数为true，表示用于判断状态的函数
[UnityEditor.MenuItem("Tools/xxx",true)]
public static bool CheckA()
{
    //是否在编译中
    return UnityEditor.EditorApplication.isCompiling;
}

交互形

这种一般就是自定窗口的形式

public class JsonGenerator : EditorWindow
{
    //入口函数，通过指令形触发
    [MenuItem("Code/JsonAnalyser")]
    public static void OpenWindow()
    {
        //开启窗口
        JsonGenerator j = EditorWindow.GetWindow(typeof(JsonGenerator)) as JsonGenerator;

        //窗口属性设置
        j.minSize = new Vector2(400, 105);
        j.maxSize = new Vector2(400, 105);
        //...
    }

    public void OnGUI()
    {
        //窗口操作逻辑
    }
}

常用的一些API

  • UI控件
    • 控件基本存放在GUILayout和EditorGUILayout中，后者就是多个自动布局而已，EditorGUILayout中有的控件GUILayout都有，反之却不成立，所以在EditorGUILayout找不到东西是可以去GUILayout看看，比如Button组件
    • 文件选择框，UnityEditor自带的APIEditorUtility.OpenFilePanel只能选择单个文件，所以可以使用第三方的插件,该插件需要开启.Net4.x
  • 状态相关的API UnityEditor.EditorApplication 如是否正在编译
  • 资源相关的API AssetDatabase 如资源刷新，资源路径
  • PlayerSetting相关的API UnityEditor.PlayerSettings 如宏定义
  • 编译相关 UnityEditor.Compilation.CompilationPipeline 如重新编译脚本
  • 工具类API EditorUtility PrefabUtility 等一系列Utility结尾的API，具体参考文档
  • 回调事件 UnityEditor.Callbacks包含事件属性，在函数上加上对应事件属性即可，可以用来build或complile时的IL注入
