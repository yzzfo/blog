为什么有时候能看到类似float3*float2

这里的乘法是逐分量的乘法，也就是说，每个分量都分别乘以对应的值。例如，float3(a,b,c) * float2(x,y) 等价于 float3(a_x,b_y,c)。

如何把NDC空间下的坐标转换成屏幕空间下的UV坐标呢？

要把NDC空间下的坐标转换成屏幕空间下的UV坐标，可以用以下公式： xuv = (xndc + 1) / 2 yuv = (yndc + 1) / 2 其中，xndc和yndc是NDC空间下的坐标，xuv和yuv是屏幕空间下的UV坐标。这个公式是把[-1, 1]范围内的NDC坐标映射到[0, 1]范围内的UV坐标。

ComputeScreenPos和NDC下UV的区别

Unity Shader中的ComputeScreenPos函数 - 知乎 (zhihu.com) ComputeScreenPos是Unity shader提供的一个内置函数，它可以根据顶点位置计算出屏幕坐标。它返回一个float4，其中xy分量是屏幕坐标，w分量是齐次坐标。要得到UV坐标，可以用xy除以w

通过ComputeScreenPos计算而来的屏幕坐标用其算出的UV和NDC空间下转换的UV有一些区别。主要是因为ComputeScreenPos考虑了不同平台上的投影参数，而NDC空间下转换的UV没有考虑这些因素

基本公式
screenPosX = ((x / w) * 0.5 + 0.5) * width  
screenPosY = ((y / w) * 0.5 + 0.5) * height

两者都是依据如下公式变换出来的
o.x = pos.x/pos.w  => [-1,1]
o.x * 0.5 + 0.5 = 0.5*pos.x/pos.w + 0.5 => [0,1]
同时乘pos.w得到如下公式 => [0,w]

ComputeScreenPos
o.x = (pos.x * 0.5 + pos.w * 0.5)  
o.y = (pos.y * 0.5 * _ProjectionParams.x + pos.w * 0.5)

NDC
o.x = (pos.x * 0.5 + pos.w * 0.5)  
o.y = (pos.y * 0.5 + pos.w * 0.5)

ComputeScreenPos 应该在Vertex中计算

在Frag中计算时发现通过用其计算的结果来算uv并不正确，但在Vertex阶段计算，然后在Frag阶段去算uv是正确的，原因在此。

计算Screen UV的例子

struct v2f  
{  
    float4 scrPos: TEXCOORD0;  
    float4 CSPos: TEXCOORD1;  
    float4 vertex: SV_POSITION;  
};  
  
v2f vert(appdata v)  
{  
    v2f o;  
    o.vertex = UnityObjectToClipPos(v.vertex);  
    o.CSPos = UnityObjectToClipPos(v.vertex);   
    o.scrPos = ComputeScreenPos(o.vertex);  
    return o;  
}  

// 以下三种return均可返回screen uv
fixed4 frag(v2f i): SV_Target  
{  
    return fixed4(i.vertex.xy/_ScreenParams.xy,0,1);  
    // _ProjectionParams.x 是否要翻转y
    return fixed4(0.5 * (i.CSPos.xy * fixed2(1, _ProjectionParams.x)) / i.CSPos.w + 0.5, 0, 1); 
    return fixed4(i.scrPos.xy / i.scrPos.w, 0, 1);
}

通过Screen Positon计算世界坐标

using System.Collections;  
using System.Collections.Generic;  
using UnityEngine;  
  
public class Scanf : MonoBehaviour  
{  
    public Material material;  
  
    void OnRenderImage(RenderTexture src, RenderTexture dst)  
    {        Camera curruntCamera = Camera.main;  
        Matrix4x4 matrixCameraToWorld = curruntCamera.cameraToWorldMatrix;  
        Matrix4x4 matrixProjectionInverse = GL.GetGPUProjectionMatrix(curruntCamera.projectionMatrix, false).inverse;  
        Matrix4x4 matrixHClipToWorld = matrixCameraToWorld * matrixProjectionInverse;  
  
        Shader.SetGlobalMatrix("_MatrixHClipToWorld", matrixHClipToWorld);  
        Graphics.Blit(src, dst, material);  
    }}

GL.GetGPUProjectionMatrix的含义 OpenGL-like（OpenGL and OpenGL ES）和Direct3D-like（Direct3D, Metal and consoles）对Clip space的表达是不一样的，这会导致projectionMatrix会不一样，这也就是Unity提供GL.GetGPUProjectionMatrix的原因，GL.GetGPUProjectionMatrix会将Camera.projectionMatrix转换为当前GPU平台对应的表达

inline half3 TransformUVToWorldPos(half2 uv)  
{  
    half depth = tex2D(_CameraDepthTexture, uv).r;  
    #ifndef SHADER_API_GLCORE  
    half4 positionCS = half4(uv * 2 - 1, depth, 1) * LinearEyeDepth(depth);  
    #else  
        half4 positionCS = half4(uv * 2 - 1, depth * 2 - 1, 1) * LinearEyeDepth(depth);  
    #endif  
    return mul(_MatrixHClipToWorld, positionCS).xyz;  
}

内置着色器变量

一些常用变量

SV_POSITION

  • xy 屏幕坐标
  • z 深度值，同ZBuffer中存储的格式一样
  • w clip space下的w，在Unity中同View Space下的z

_ProjectionParams

  • x 用于判断y是否需要翻转

_ScreenParams

  • xy 屏幕大小
