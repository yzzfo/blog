CloneBot V2是一个Telegram的机器人，用于转存Google Driver的资源

安装方式

获取共享盘

https://t.me/MSGuite_SD_Creator_Bot

配置文件

[General]
path_to_gclone = gclone

telegram_token = 
user_ids =
group_ids = -1

gclone_para_override =

将配置文件存到GitHub Gist中

Docker-Compose

    clonebot:
      image: ghcr.io/thecaduceus/clonebot-v2:main
      container_name: clonebot
      networks:
        - main    
      tty: true
      environment:
        - CONFIG_FILE_URL= #gist raw链接
      restart: unless-stopped

创建Service Accounts，让机器人正常运行

方式一

  1. 参照 https://alist-doc.nn.ci/docs/driver/googledrive/#%E5%9B%BE%E6%96%87%E6%95%99%E7%A8%8B%E8%A7%86%E9%A2%91%E7%89%88%E5%9C%A8%E4%B8%8B%E9%9D%A2 文章创建API和OAuth
  2. 在选项API和服务->凭据->服务账号中添加一个新的账号，添加后点击操作->管理密钥对密钥进行设置在结束时选择保存为json。
  3. 参照该文章设置Google Group https://github.com/TheCaduceus/CloneBot_V2#-adding-in-google-group ,用于赋予操作网盘的权限
  4. 上面获取的json就是一个用于操作的sa账户
  5. 将其打包到accounts.zip，发送至机器人并附带/sa即添加成功了账号

方式二

参考这篇官方文章 https://github.com/TheCaduceus/CloneBot_V2#-service-accounts ,我在按照里面的操作进行是出现机器人读不到共享磁盘的问题，可能是由于当时操作出现网络问题，故没有继续尝试下去，后改为方式一操作成功。
