win中的eol问题.
Posted on 2023-12-04 :: Tags: dev

Table of Contents

  • core.autocrlf

在Windows系统下文件结尾默认为/r/n即CRLF，但Linux下却是/n即LF这点差异在一般的文本文件中到没什么问题会出现，但对于shell脚本，由于/r/n的出现，会导致其无法运行，对于简单的复制脚本到Linux，可以使用dos2unix命令来对文件进行转换，但如果使用devContainer来开发程序那就不可避免的要进行一番设置了。

  1. 将vscode设置为LF结尾，在设置中搜索eol将其设置为\n即可。
  2. git仓库进行reset，使其回归\n结尾
# https://stackoverflow.com/questions/48692741/how-can-i-make-all-line-endings-eols-in-all-files-in-visual-studio-code-unix
git config core.autocrlf false

git rm --cached -r .         # Don’t forget the dot at the end

git reset --hard

这样在devContainer中打开文件就说LF结尾的了。

core.autocrlf

core.autocrlf这个值为true的时候 git仓库中存的都是LF结尾的文件，当拉取的时候git会判断平台将文件的结尾进行对于更改。 为了使win下也是LF，所以需要将他设为false，这样就能使用原始文件结尾了。
