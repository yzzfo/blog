在之前学习Shader时一直有一个误解，认为SV_POSITION在Frag阶段依旧是裁剪坐标系，于是在手动计算深度缓存时出现了一些问题，原因在此。

通常来说在NDC空间下z值(-1,1)映射到depth = 0.5*ndcz+0.5，即ZBuffer中的值。在Frag阶段只需通过SV_POSITION即可获取当前像素所在的depth值。

// depth 值在Unity中由 SHADER_API_GLCORE 宏做区分
// uv 为screenUV
// clip space下的w坐标其实就是viewspace下的z值
#ifdef SHADER_API_GLCORE  
	half4 positionCS = half4(uv * 2 - 1, depth * 2 - 1, 1) * LinearEyeDepth(depth);
#else  
    // 在Win下走的这
	half4 positionCS = half4(uv * 2 - 1, depth, 1) * LinearEyeDepth(depth);        
#endif

也可以通过Unity提供的API在Vertex阶段来计算深度
UnityCG.cginc

#define COMPUTE_EYEDEPTH(o) o = -mul( UNITY_MATRIX_MV, v.vertex ).z
#define COMPUTE_DEPTH_01 -(mul( UNITY_MATRIX_MV, v.vertex ).z * _ProjectionParams.w)

在顶点阶段调用上面的任意函数获取深度值，之后到片元着色器去使用，在片元着色器通过如下函数对_CameraDepthTexture中的值解码

// Z buffer to linear 0..1 depth (0 at eye, 1 at far plane)
inline float Linear01Depth( float z )
{
    return 1.0 / (_ZBufferParams.x * z + _ZBufferParams.y);
}
// Z buffer to linear depth
inline float LinearEyeDepth( float z )
{
    return 1.0 / (_ZBufferParams.z * z + _ZBufferParams.w);
}

使用案例

Vertex

通过顶点计算深度值

struct v2f  
{  
    float2 uv: TEXCOORD0;  
    float4 screenPos: TEXCOORD1;  
    float4 vertex: SV_POSITION;  
    float depth:DEPTH;  
};

v2f vert(appdata v)  
{  
    v2f o;  
    o.vertex = UnityObjectToClipPos(v.vertex);  
    o.screenPos = ComputeScreenPos(o.vertex);
    o.depth = COMPUTE_DEPTH_01(o.vertex);
    o.uv = v.uv;  
    return o;  
}

Frag

获取当前ZBuffer中的值的方式

_CameraDepthTexture
Camera.main.depthTextureMode |= DepthTextureMode.Depth;

这样，相机就会生成深度纹理，并且你可以在Shader中访问它。如果是延迟渲染，则不需要手动设置，因为相机默认就会生成深度纹理。

// 获取当前像素在屏幕上的坐标
float2 screenUV = i.screenPos.xy / i.screenPos.w;
// 从_CameraDepthTexture中采样当前像素对应位置处的深度值
float depth = SAMPLE_DEPTH_TEXTURE(_CameraDepthTexture, screenUV);
// 将深度值转换为视空间下距离摄像机远近程度
float linear01Depth = Linear01Depth(depth);

Note
_CameraDepthTexture生成的Depth是通过"LightMode" = "ShadowCaster"的Pass来渲染出来的，在Surface Shader中通常只需要加入addshadow即可，若Shader中两者都不存在，若Fallback中的Shader有ShadowCaster也会渲染出Depth。

也存在一种不需要任何设置就能获取的全局属性_LastCameraDepthTexture即上一次的深度信息，使用方式同上。

_CameraDepthNormalsTexture
Camera.main.depthTextureMode |= DepthTextureMode.DepthNormals;

// UnityCG.cginc
// 解析函数
inline void DecodeDepthNormal( float4 enc, out float depth, out float3 normal )  
{  
	depth = DecodeFloatRG (enc.zw);  
	normal = DecodeViewNormalStereo (enc);  
}

通过该模式，你可以在Shader中访问到Depth以及Normal信息

// 获取当前像素在屏幕上的坐标
float2 screenUV = i.screenPos.xy / i.screenPos.w;
// 从_CameraDepthNormalsTexture中采样当前像素对应位置处的数据
float4 enc = SAMPLE_DEPTH_TEXTURE(_CameraDepthNormalsTexture, screenUV);
float3 normal = DecodeViewNormalStereo(enc);

Note
_CameraDepthNormalsTexture使用的ShaderReplacement进行生成，ShaderReplacement通过匹配RenderType进行替换操作，默认使用Internal-DepthNormalsTexture.shader进行替换操作，若要更改需要在Unity中进行相关设置，如下图。

一些疑问

在Unity中为什么NDC空间下Z值越近越大？
在Unity中，NDC空间下的z值从大到小是因为Unity使用了反向z(Reversed-Z)来处理z fighting问题。反向z先是把摄像机的矩阵改了，然后自动把你的shader的ztest creater和ztest less翻转，深度测试本身就没问题了。
