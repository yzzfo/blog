  在学习Shader时经常会看到这么一条注意事项少用if else语句，通常的原因是在着色器中使用条件语句（如if）可能会影响效率，因为它们会破坏着色器的并行性。如果你想根据某个值的大小来选择不同的输出，你可以使用step函数，它会返回0或1，然后你可以用它来做插值或混合。step函数比if语句更高效，因为它没有分支。

为什么Step比 If 快

step函数的实现可能会根据不同的平台和语言有所差异，但是一种常见的实现方法是使用比较运算符和逻辑运算符。例如，如果你想实现一个返回0或1的step函数，你可以这样写：

float step(float edge, float x) {
  return (x < edge) ? 0.0 : 1.0;
}

这里的?是一个三元运算符，它相当于一个简单的if-else语句，但是它不会产生分支。你可以把它理解为：

float step(float edge, float x) {
  if (x < edge) {
    return 0.0;
  } else {
    return 1.0;
  }
}

但是这种写法就会有分支，因为它会根据条件选择不同的返回值。而三元运算符只是一个表达式，它会同时计算两个可能的返回值，然后根据条件选择其中一个作为结果。这样就避免了分支。

三目运算符为什么没有分支，这个问题的答案可能会取决于编译器和CPU的具体实现。一种可能的解释是，三目运算符是一个表达式，它可以被优化为一个单一的指令，而不需要跳转或分支。另一种可能的解释是，三目运算符可以让CPU更容易预测条件的结果，从而减少分支预测错误的代价。无论哪种情况，三目运算符都比if语句更高效，因为它避免了不必要的分支。
