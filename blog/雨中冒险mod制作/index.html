最近开始玩起来了雨中冒险2这款游戏，发现这款游戏是使用Unity制作的而且还支持Mod于是就打算来修改修改游戏内的逻辑。

雨中冒险Mod环境安装

  1. 下载 BepInExPack 并解压至游戏根目录
  2. 安装 R2API 放在**Risk of Rain 2/Bepinex/plugins/**目录下

新建Mod项目

  1. 使用Rider创建新的C# Class Library项目，Framework选择netstandard 2.0，SDK选择3.0以上。
  2. 创建插件
     using System;
 using System.Collections;
 using System.Collections.Generic;
 using System.Reflection;
 using BepInEx;
 using EntityStates;
 using RoR2;
 using GenericSkill = On.RoR2.GenericSkill;
 using HarmonyLib;
 using IL.RoR2.Skills;
 using R2API.Utils;
 using UnityEngine;
 using DelayBlast = On.RoR2.DelayBlast;

 namespace MyMod
 {
     //GUID -> www.hello.app /config下的配置与GUID同名
     //Name -> 插件名
     //Ver -> 1.0 版本号
     [BepInPlugin("www.hello.app", "Hello", "1.0")]
     public class Mod : BaseUnityPlugin  //插件基类
     {
         public Mod()
         {
             //Hook 原本的逻辑
             //RoR2.ChestBehavior.ItemDrop为原本的函数
             //On.RoR2.ChestBehavior.ItemDrop为原本函数调用事件，将原本函数Hook到该事件中
             //开箱是随机增加掉落数量
             On.RoR2.ChestBehavior.ItemDrop += ((orig, self) =>
             {
                 float per = UnityEngine.Random.Range(0f, 1f);
                 int add = 0;
                 if (per < 0.005f)
                 {
                     add = 10;
                 }
                 else if (per < 0.03f)
                 {
                     add = 5;
                 }
                 else if (per < 0.13f)
                 {
                     add = 3;
                 }
                 else if (per < 0.33f)
                 {
                     add = 2;
                 }
                 else if (per < 0.63f)
                 {
                     add = 1;
                 }
                 else if (per < 0.631f)
                 {
                     add = -1;
                 }

                 var filed = self.GetType().GetField("dropCount", BindingFlags.NonPublic | BindingFlags.Instance);
                 int ordCount = (int) filed.GetValue(self);
                 var tarInt = ordCount + add;
                 if (tarInt < 0)
                 {
                     tarInt = 0;
                 }

                 filed.SetValue(self, tarInt);
                 orig(self);
             }
     }
 }
    1. Build该工程，在Risk of Rain 2/Bepinex/plugins/目录中创建文件夹并将Build后的dll文件放入其中，至此Mod安装完毕。

断点调试Mod

  1. 安装dnSpy
  2. 查看游戏使用的Unity版本
  3. 查看Risk of Rain 2\MonoBleedingEdge\EmbedRuntime\mono-2.0-bdwgc.dll PE头中的时间戳
    3.1 方法一：使用dnSpy查看时间戳，该时间戳为本地时间
    3.2 方法二：使用umpatcher查看时间戳，见第七步中的使用
  4. 拉取dnSpy-Unity-mono并切换到dnSpy分支
  5. 拉取Unity-Mono并找到对应Unity版本的的分支
    例如v5.4.3, 分支名为unity-5.4， 结尾带-mbe使用 .NET 4.x assemblies 找到和时间戳最接近的提交，复制hash
  6. 编译umpatcher
    在dnSpy-Unity-mono中的找到src\umpatcher\umpatcher.sln使用visual studio打开并Build即可
  7. 使用umpatcher
     #查看时间戳
 umpatcher --timestamp "C:\path\to\the\correct\version\mono.dll"

 #生成前确保Unity-mono和dnSpy-Unity-mono都是纯净工程，即git statue后没有提示任何文件，否则将导致失败并且程序会执行很长的时间，由于文件过多。
 #生成指定版本的dnSpy-Unity-mono工程
 #版本 为游戏的Unity版本
 #hash 为之前第五步复制的hash
 umpatcher 5.4.3 aa8a6e7afc2f4fe63921df4fe8a18cfd0a441d19 "C:\path\to\Unity-mono" "C:\path\to\dnSpy-Unity-mono"
 #2019.2之后似乎都是.net4了，需要加上-mbe否则会走原先的生成逻辑
 umpatcher 2019.4.26-mbe 5c72e2513122ba54eb1dcaef03df123ca1a2809b "C:\path\to\Unity-mono" "C:\path\to\dnSpy-Unity-mono"
  8. 编译mono-2.0-bdwgc.dll
    打开dnSpy-Unity-mono目录找到dnSpy-Unity-mono-v2019.x-V40.sln，用Visual Studio打开，选中里面的项目2019.4.26-mbe进行生成即可获得mono-2.0-bdwgc.dll
  9. 用获得的mono-2.0-bdwgc.dll将游戏中的mono-2.0-bdwgc.dll文件进行替换
  10. 用dnSpy进行运行游戏
    10.1. 选择调试->开始调试
    10.2. 进行如下设置
  11. 拖入想调试的dll，断点即可
