Compose

DMS_GITHUB_URL="https://raw.githubusercontent.com/docker-mailserver/docker-mailserver/master" wget "${DMS_GITHUB_URL}/compose.yaml" wget "${DMS_GITHUB_URL}/mailserver.env"

生成相关数据

生成用户

docker exec -ti mailserver setup email add name@hostname

生成DKIM

docker exec -ti mailserver setup config dkim keysize 2048

文件在 `dms/config/opendkim/keys/mail.txt

配置 DNS

  • 添加一个 A 记录，名称为 mail, 指向自己服务器的公网 ip
  • 添加MX记录， 指向邮件服务器域名 mail.[yourdomain], 优先级设置为10
  • 添加TXT SPF 记录, 可以简单使用: v=spf1 mx -all
  • 添加 TXT _dmarx 记录, Name 设置为 _dmarc, Content 设置为 v=DMARC1; p=none; pct=100; rua= my@email.com
  • 添加 TXT DKIM 记录, Name 设置为 mail._domainkey, Content 设置为类似如下内容 textv=DKIM1; h=sha256; k=rsa; p=xxxxxxxxxxxxxxxxxxxxxxxxxxxxx，内容来自 mail.txt `

TLS

修改 mailserver.env 文件中如下内容，可参考 https://docker-mailserver.github.io/docker-mailserver/edge/config/security/ssl/

SSL_TYPE=manual 
SSL_CERT_PATH=/tmp/dms/custom-certs/fullchain.pem 
SSL_KEY_PATH=/tmp/dms/custom-certs/privkey.pem

重启服务即可

SPOOF_PROTECTION

在 mailserver 中存在该选项，默认开启，会导致直接使用邮箱地址为用户名出现拒绝发件的情况。

PTR 反向解析

在 VPS 提供商出进行设置，以 Servarica 为例，其在 Hostname 选项的下方提供了一个 PTR 的按钮可进行修改。

客户端

在 IOS 或者 Windows 自带的邮件应用中，IMap 协议默认不会发送密码，所以需要在相关的设置中单读设置一下密码才能使用。
