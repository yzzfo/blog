在Docker中禁止容器使用Swap.

环境

Ubuntu22.04
Docker 20.10.21

启用CGroup中的Swap控制功能

  1. 在/etc/default/grub文件中编辑或者加入GRUB_CMDLINE_LINUX="cgroup_enable=memory swapaccount=1"
  2. update-grub
  3. reboot
  4. cat /proc/cmdline 进行验证

在Docker-Compose中禁止Swap

# 以下配置为version 2.4时的配置，3.x可能并不适用
halo:
   restart: always
   image: halohub/halo
   container_name: halo
   networks:
     - main
   environment:
     - VIRTUAL_PORT=8090
     - VIRTUAL_HOST=hostname  # 你的博客地址（务必修改）
     - LETSENCRYPT_HOST=0.0.0.0 # 你的博客地址 （务必修改）
     - LETSENCRYPT_EMAIL=email # 证书所有者的邮箱，快过期时会提醒（务必修改）
     - MAX_UPLOAD_SIZE=10m
     - JVM_XMS=256m
     - JVM_XMX=256m
   mem_swappiness: 0 #内存交换频率 0表示内核积极地避免尽可能长时间地交换
   mem_limit: 512m	#内存限制
   memswap_limit: 512m	#与mem_limit设置相同时意为禁用swap，不设置swap会使用当前内存的2倍memory
   volumes:
     - ./halo:/root/.halo
