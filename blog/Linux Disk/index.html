  最近在使用exis时经常要使用到关于磁盘操作的指令，故在此记录一下这些指令的使用场景和用法。

运行环境

  1. exsi 7.0.3

  2. ubuntu-22.04-live-server-amd64

  3. 安装ubuntu时选择了Set up this disk as an LVM group

创建新的分区

在使用exsi时，如果我们后续给ubuntu新增一块硬盘时需要在ubuntu中对其分区格式化操作后才能使用，其具体步骤如下。

  1. fdisk -l 找到形如下边的Disk，这既是刚刚挂在上去的磁盘

    Disk /dev/sda: 400 GiB, 429496729600 bytes, 838860800 sectors
Disk model: Virtual disk    
Units: sectors of 1 * 512 = 512 bytes
Sector size (logical/physical): 512 bytes / 512 bytes
I/O size (minimum/optimal): 512 bytes / 512 bytes
Disklabel type: dos
Disk identifier: 0x280990c6

  2. fdisk /dev/sda 为磁盘创建分区，具体操作可参考该博文Ubuntu如何新建分区

    创建后使用fdisk -l查看可以看到新增的Device

    Device     Boot Start       End   Sectors  Size Id Type
/dev/sda1        2048 838860799 838858752  400G 83 Linux

  3. partprobe 同步分区表至系统

  4. mkfs.ext4 /dev/sda1 格式化分区

  5. mkdir /mnt/disk 新建挂载目录

  6. 添加/dev/sda1 /mnt/disk ext4 defaults 0 1至/etc/fstab文件

    # /etc/fstab: static file system information.
#
# Use 'blkid' to print the universally unique identifier for a
# device; this may be used with UUID= as a more robust way to name devices
# that works even if disks are added and removed. See fstab(5).
#
# <file system> <mount point>   <type>  <options>       <dump>  <pass>
# / was on /dev/ubuntu-vg/ubuntu-lv during curtin installation
/dev/disk/by-id/dm-uuid-LVM-UG7dOVca40ecQwEntS5Bm2YkYgCCJ1iJdFHVKxx0OBv16rf3wbOadnwSVhRHa4Xl / ext4 defaults 0 1
# /boot was on /dev/sda2 during curtin installation
/dev/disk/by-uuid/c34aad53-5edb-4135-997d-68a1e7fac8ca /boot ext4 defaults 0 1
/dev/sda1       /mnt/disk       ext4    defaults        0       1

  7. mount -a挂载全部文件

  8. df -h查看挂载情况，出现def/sda1即成功挂载

    Filesystem                         Size  Used Avail Use% Mounted on

tmpfs                              2.0G     0  2.0G   0% /run/qemu
/dev/sdb2                          1.7G  125M  1.5G   8% /boot

扩容现有磁盘

  在ubuntu编译openwrt时，编译到最后出现磁盘空间不足的报错，于是需要给现在挂载的磁盘进行扩容操作，此处假设扩容2G。

  1. 在exsi给指定磁盘分配更大的空间

  2. 分区大小调整

    1. 使用gparted调整分区大小，具体操作参考Ubuntu server在ESXi下进行磁盘无损扩容

    2. 使用parted扩容

      parted /dev/nvme0n1
GNU Parted 3.2
Using /dev/sda
Welcome to GNU Parted! Type 'help' to view a list of commands.
(parted)

print    #打印

Model: VMware, VMware Virtual S (scsi)
Disk /dev/nvme0n1: 62.3GB
Sector size (logical/physical): 512B/512B
Partition Table: gpt
Disk Flags:

Number  Start   End     Size    File system  Name  Flags
 1      17.4kB  1049kB  1031kB                     bios_grub
 2      1049kB  538MB   537MB   fat32              boot, esp
 3      538MB   34.4GB  33.8GB                     lvm


resizepart 3 100%
quit

    3. 使用gdisk调整分区大小

      # 此处假设分区为3
gdisk /dev/nvme0n1
d    #删除原本的分区
n    #创建新的分区
w    #将结果写入

partprobe   #应用gdisk的写入

pvdisplay /dev/nvme0n1p3    #此时应该还是显示的以前的大小

pvresize /dev/nvme0n1p3  #通知更新pv容量

  3. lvdisplay 显示逻辑磁盘，此处可以知道逻辑磁盘路径为/dev/ubuntu-vg/ubuntu-lv

      LV Path                /dev/ubuntu-vg/ubuntu-lv

  4. pvdisplay 查看物理磁盘路径

      --- Physical volume ---
  PV Name               /dev/sdb3
  VG Name               ubuntu-vg

  5. vgs 查看vg空间是否满足LV扩容

      VG        #PV #LV #SN Attr   VSize   VFree
  ubuntu-vg   1   1   0 wz--n- <18.25g 2.0g

  6. 扩容

    1. lvextend -L +2GB /dev/ubuntu-vg/ubuntu-lv

    2. lvextend -L +100%FREE /dev/ubuntu-vg/ubuntu-lv

  7. resize2fs /dev/ubuntu-vg/ubuntu-lv重新加载逻辑卷

  8. df -h查看结果即可
