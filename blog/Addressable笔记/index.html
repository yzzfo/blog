增量更新

配置

  1. 修改Addressable Asset Settings中的Catalog，使其支持服务端清单，Only update catalogs manually 选项用于禁止自带的自动更新功能。

  2. 为资源组设置远程模式，其中Use Asset Bundle Cache用于标记该bundle下载后会缓存到设备上，Prevent Updates勾选后则意为该资源为静态资源，对该组的修改后续将进行增量更新，反之为全量更新。

  3. 配置服务器地址，其中RemoteLoadPath对应bundle所在的文件夹，修改Play Mode Script为Use Existing Build用以使用真实环境测试。

打包

  1. Build第一个包
  2. 修改了某个Group中的资源后，进行增量更新，这里只有勾选了Prevent Updates组中的物体才会出现，填入一个新的Group名即增量包名，点击Apply即会创建出一个新的组，之后选择更新新包即可打出新的资源。

代码

IEnumerator DoUpdateAddressadble()  
{  
    // 这一句话包含了CheckForCatalogUpdates这个副作用，所以注释调  
    // 如果之前没调用InitializeAsync，后续别的函数执行时也会自己重新调一遍InitializeAsync  
    // AsyncOperationHandle<IResourceLocator> initHandle = Addressables.InitializeAsync();    // yield return initHandle;  
    // 检测更新  
    var checkHandle = Addressables.CheckForCatalogUpdates(false);  
    yield return checkHandle;  
    if (checkHandle.Status != AsyncOperationStatus.Succeeded)  
    {        Debug.LogError("CheckForCatalogUpdates Error\n" + checkHandle.OperationException.ToString());  
        yield break;  
    }  
    if (checkHandle.Result.Count > 0)  
    {        Debug.Log("有更新");  
        var updateHandle = Addressables.UpdateCatalogs(checkHandle.Result, false);  
        yield return updateHandle;  
  
        if (updateHandle.Status != AsyncOperationStatus.Succeeded)  
        {            Debug.LogError("UpdateCatalogs Error\n" + updateHandle.OperationException.ToString());  
            yield break;  
        }  
        // 更新列表迭代器  
        List<IResourceLocator> locators = updateHandle.Result;  
        foreach (var locator in locators)  
        {            var keys = locator.Keys.ToList();  
            // 获取待下载的文件总大小  
            // 注意不能传入IList接口，如果是List需要强转为(IEnumerable)才行  
            // 因为当前版本有问题存在三种签名  
            // GetDownloadSizeAsync(IList<object> keys) [Obsolete] 这导致IList直接走object接口了，进而报错  
            // GetDownloadSizeAsync(IEnumerable keys)  
            // GetDownloadSizeAsync(object key)            var sizeHandle = Addressables.GetDownloadSizeAsync((IEnumerable)keys);  
            yield return sizeHandle;  
            if (sizeHandle.Status != AsyncOperationStatus.Succeeded)  
            {                Debug.LogError("GetDownloadSizeAsync Error\n" + sizeHandle.OperationException.ToString());  
                yield break;  
            }  
            long totalDownloadSize = sizeHandle.Result;  
            Debug.Log("download size : " + totalDownloadSize);  
            Debug.Log("download size : " + totalDownloadSize);  
            if (totalDownloadSize > 0)  
            {                // 下载  
                var downloadHandle =  
                    Addressables.DownloadDependenciesAsync(keys, Addressables.MergeMode.Union, false);  
                while (!downloadHandle.IsDone)  
                {                    if (downloadHandle.Status == AsyncOperationStatus.Failed)  
                    {                        Debug.LogError("DownloadDependenciesAsync Error\n" +  
                                       downloadHandle.OperationException.ToString());  
                        yield break;  
                    }  
                    // 下载进度  
                    float percentage = downloadHandle.PercentComplete;  
                    Debug.Log($"已下载: {percentage}");  
                    yield return null;  
                }  
                if (downloadHandle.Status == AsyncOperationStatus.Succeeded)  
                {                    Debug.Log("下载完毕!");  
                }            }        }    }    else  
    {  
        Debug.Log("没有检测到更新");  
    }}

  3. 在 Simulate Groups 模式下需要使用如下代码来修改速度，一下设置虽然说是 ProjectConfigData，但没有在 Project 相关配置中找到，只在 Library\com.unity.addressables\catalog.json 这个文件中搜索到了相关配置，所以在设置后第一次运行不生效，需要重新生成 catalog.json 才会生效。

	ProjectConfigData.LocalLoadSpeed = 1024 * 1024 * 10;  
	ProjectConfigData.RemoteLoadSpeed = 1024 * 1024 * 10;  
#endif

Catalogs

  1. 当关闭Only update catalogs manually时Addressables.InitializeAsync();会更新Catalogs文件
  2. 手动调用Addressables.UpdateCatalogs()相当于关闭Only update catalogs manually`干的事
  3. 当勾选Only update catalogs manually时，直接调用Addressables.CheckForCatalogUpdates(false);时Result将包含所有内容。
  4. 更新检查仅是对比Catalogs文件，所以当CheckForCatalogUpdates后去下载对应的文件中途断开后，重新进入CheckForCatalogUpdates将无法获取到新的内容，因为本地Catalogs文件已经为最新的了。
  5. 如果中途断开，若之后用到了被更新的文件，Addressables将去获取新的文件。
以下代码等效于Addressables.InitializeAsync()勾选`Only update catalogs manually`时所作的事
var checkHandle = Addressables.CheckForCatalogUpdates();  
yield return checkHandle;  
if (checkHandle.Status == AsyncOperationStatus.Succeeded && checkHandle.Result.Count > 0)  
{  
    var updateHandle = Addressables.UpdateCatalogs();  
    yield return updateHandle;  
}

资源载入

在Addressable中有两种加载方式，一种是LoadAssetAsync另一种是InstantiateAsync，他们的区别在于，LoadAssetAsync后必须Release否则引用计数会一直存在，InstantiateAsync则是当切换场景时，若实例化的对象被销毁了对应的引用计数也会消失。
在Addressable类这个层，LoadAssetAsync和InstantiateAsync均可以使用Release进行销毁，ReleaseInstance仅添加了实例判空检测。
之所以InstantiateAsync在切换场景是会判断引用计数是可以参考Library/PackageCache/com.unity.addressables@1.21.9/Runtime/ResourceManager/ResourceProviders/InstanceProvider.cs可以看到对于实例化物体都有一个字典将其记录了。

Release注意点

// 以下这段代码是错误的
List<TextAsset> assets = new List<TextAsset>();  
yield return Addressables.LoadAssetsAsync<TextAsset>("config", (x) =>  
{  
    assets.Add(x);  
});  
Addressables.Release(assets);

// 这种才是正确的，Release只能销毁响应结果的引用
var configHandle = Addressables.LoadAssetsAsync<TextAsset>("config",null);  
yield return configHandle;
Addressables.Release(configHandle);

注意点

  1. Non-Recursive Dependency Calculation：开启该选项可以提高构建速度同时减少运行内存的占用，但资源存在循环依赖时会导致加载失败。不建议开启。
  2. 场景切换时死锁问题
  3. 切换平台后, Editor 下读取路径出现问题，解决方法是清除一下缓存。
  4. 切换 Android 后出现 TLS 错误，在开发模式下 Edit -> Project Settings -> Player -> Other Settings -> Configuration -> Allow downloads over HTTP 可以在这设置一下。
  5. 读取 Built in date 中的文件时文件Key 没有后置名，同 Resource. Load 一样不需要文件后缀名，截至 1.21 版本在 Group 界面下复制 Resources 文件夹下的 Key 时还是会多出后缀名，去除后缀名才是真正的 Key。具体参考 ResourceLocationMap.cs 文件中的 Locate 函数，在此处断点可看到具体的 Key 值。
