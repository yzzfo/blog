  在游戏开发时人们经常会实现一些网络传输方案，在使用tcp时，由于其的确认机制及拥塞控制通常会导致网络波动，而udp则过于简单仅包含一份数据传输时所需的基本信息，这也使得udp可以快速传递信息。对于游戏而言我们希望的传输协议应该具有一下几点:

  1. 低延迟
  2. 尽可能的可靠
  3. 有序

为了实现这一点通常就是使用udp来实现一个简易的tcp，剔除tcp中一些繁杂的检测机制，在传递信息时我们允许一些丢包而不是在那一直等待，例如在同步玩家位置这类并非十分关键的数据时，中间丢失几帧的信息是可以允许的。

tcp在传递数据时采用的滑动窗口来保证数据的可靠及有序，在出现丢包时会触发重传机制，这也就导致了延迟的增大，这时就需要使用FEC技术。

FEC即Forward Error Correction的缩写，中文为前向纠错，FEC是利用数据进行传输冗长信息的方法,当传输中出现错误,将允许接收方修复数据。

XOR FEC

在二进制情况下异或操作有如下特性,其实本质就是不进位加法

  1. 1为取反
  2. 0保持不变
  3. 0^1^0^1^1^X 当1的总数为奇数时结果为1偶数为0

所以当我们有N组数据时，通过将N组数据进行异或操作，即可获得一份纠错数据，例如:

0101
1111 => 1010
0000

更具第3点,此时1010可以看作是数据奇偶的特征
若奇偶性不变则 => 0
改变则 => 1
由于奇偶性不变时其异或的结果也与特征结果一致可以推出X^X=0，改变时则与其相X^1，则我们能直接通过异或运算得出丢失的数据。

例如此时我们任意丢失一组数据0101，对余下的数据进行异或操作可以得到1111，1111^1010 = 0101

该方法的好处在于当只丢失任意一份数据时，我们都能找回来，即增加1/N的冗余数据即可保护剩下的N份数据。
