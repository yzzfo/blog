颜色空间

Gamma

  由于人眼对于暗色的变化更为敏感，所以在精度有限的情况下会分配更多的有效值用于描述暗部，于是引申出了Gamma颜色空间。通过 y=x^0.45 在x = 0-1范围内，将暗部值精度提升，本质上是对图片进行增亮操作。通常显示器在显示时会进行Gamma解码操作，即y=x^2.2 本质是进行 变暗 操作。

Linear

线性空间就是Gamma值为 1 的情况，也就没所谓的编码解码的必要了。

PNG

  对于PNG这种文件来说，并不关心Gamma或者Linear，他只管保存颜色的数据，至于该颜色数据读取后是该Gamma校正还是直接使用由使用者来决定。

  1. 对于颜色(0.5,0.5,0.5)，若使用Gamma编码则其在PNG中的值为(0.73,0.73,0.73)，若使用Linear则依然是(0.5,0.5,0.5)
  2. 在PNG的元数据中可以查看Gamma值，但该值并不是必须保存的，所以在没有Gamma值时并不能判断该图片是否进行过Gamma编码

sRGB

1996年，微软和惠普一起开发了一种标准sRGB色彩空间。这种标准得到许多业界厂商的支持。sRGB对应的是Gamma0.45所在的空间

Unity中的颜色空间

Linear

  1. 在Linear Space下，图片导入时若勾选sRGB，则在Shader中使用该图片时会自动进行Gamma校正，即对图片进行变暗操作，使颜色值回归线性，即 (0.73,0.73,0.73) => (0.5,0.5,0.5)
  2. Shader输出颜色时，Unity会自动进行Gamma编码，即假设当前frag输出(0.5,0.5,0.5)，此时若将其保存到Texture时可以发现存储的值为(0.73,0.73,0.73)，即在Linear Space下，Unity输出的图片为sRGB。

Gamma

  1. 在Gamma Space下sRGB选项将不会生效，将会按照实际值传入Shader中，若要解码则需要自己在Shader中进行相应操作。
  2. Shader输出时并不会自动转换为sRGB，即frag输出(0.5,0.5,0.5)，Texture中存储的也为(0.5,0.5,0.5)。

Texture Setting

  1. Alpha is Transparency
    • 如果指定的 Alpha 通道为透明度 (Transparency)，则启用此属性可扩充颜色并避免边缘上的过滤瑕疵。
    • 由于勾选后可能会改变图片原本的RGB，所以需要更具情况选择是否启动。
  2. Format
    • 对于Texture的不同设置可以通过断点来具体看看不同设置对应的属性
    • https://docs.unity3d.com/ScriptReference/Experimental.Rendering.GraphicsFormat.html
    • 例如RGB16其graphicsFormat: B5G6R5_UNormPack16 ，所以他并不支持sRGB选项，若要正常使用则需要切换格式或者在Shader中手动解码

Color Property

在Shader中，Color属性是以sRGB模式传递的，所以在Linear空间下他会自动De-Gamma,SL-PropertiesInPrograms

When using Linear color space, all material color properties are supplied as sRGB colors, but are converted into linear values when passed into shaders.

For example, if your Properties shader block contains a Color property called “MyColor“, then the corresponding ”MyColor” HLSL variable will get the linear color value.

For properties that are marked as Float or Vector type, no color space conversions are done by default; it is assumed that they contain non-color data. It is possible to add [Gamma] attribute for float/vector properties to indicate that they are specified in sRGB space, just like colors (see Properties).  

    // linear空间下
    Properties
    {
        // 这里的Color为Gamma值
        _Color ("Color", Color) = (0.5,0.5,0.5)

        // 默认情况下为线性值，若声明[Gamma]则会被de-gamma
        _VecColor ("VecColor", Vector) = (0.5,0.5,0.5,1)
    }

    ...
    ...

    Pass
    {
        // de-gamma
        //此时这里的默认值为(0.217,0.217,0.217)
        float4 _Color;  

        // 由于是Vector类型，这里依旧是(0.5,0.5,0.5,1)，除非上面声明[Gamma]了
        float4 _VecColor;  
    }

导入的层级图失效

  1. 层级图 每一层对应的R=index x 0.1 G=0.5
  2. shader中根据G>=0.49，Ceil(R x 10)来判断是否属于层级，以及获取层级
    失败的原因在于，用于导出层级图的Unity项目使用的Gamma Space，而主工程使用的是Linear，new Texture时默认为sRGB，导致图片整体变暗，所以G>=0.49判断条件已然不满足，且后续层级获取也将出现问题，正确方法是new Texture时将linear参数设置为true。 设置为RGB16时图片变正常了？
    因为在Unity中RGB16不支持sRGB选项，所以使用时自动采用了PNG中的值输入到Shader中，所以正常了。
