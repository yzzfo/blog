Isometric

设置渲染轴

  1. 在SRP下Edit > Project Settings > Graphics 设置Transparency Sort Axis 为 (0, 1, 0)
  2. 在URP下需要进行如下设置
  3. Create > Rendering > URP Asset (with 2D Renderer)创建2D渲染器
  4. 在Project Settings > Graphics中使用2D的渲染管线
  5. 查看相机Rendering > Renderer属性中是否选择到了目标文件，若没有则到Project Settings > Quality中查看当前是否有旧的Render Pipeline Asset设置残留，比如用URP 3D 模板创建的项目会更预先生成几份不同画质的配置在此处设置，会导致第二步中的配置被该处覆盖，可根据项目质量自行在该出修改。
  6. 在Renderer2D文件中修改轴

设置图片资源以及网格比例

如上图所示，在Isometric中，一个物体本质是由一个基础的面从上自下刷出来的，计算比例时便是拿这个基础的面，在这个图片中其比例为(128 : 66) => (1 : 0.515)

设置CellSize

在Grid组件上，Size采用Unity空间下的长度为单位，若要将上图的映射到一个格子内，则Size需要满足比例(1 : 0.515)才能保证平铺，此处以(2 : 1.03)为例子

设置图片所对应的Unit大小

为了铺满(2 : 1.03)则Pixels Per Unit设置为128/2=64，使得在该图片中每128px对应Unity空间中的长度2即可。

关于Pattern

Pattern只不过是一个预制体+Editor，当创建完后想修改基本参数如Cell Layout则可以到预制体中修改，Pattern中设置并不会影响到实际场景中，显示仅作为在该种布局下的预览而已，例如在Pattern中设置Layout为Rectangle但是场景中是Isometric也并不影响。

计算坐标

var worldPos = Camera.main.ScreenToWorldPoint(Input.mousePosition);
worldPos.z = Tilemap.transform.position.z; //需要保持z一致，虽然是2D的，若不一致则计算出错
var cell = Tilemap.WorldToCell(worldPos);  

// 该方法计算出的是Tile Anchor的位置
// var cellcenter = Tilemap.GetCellCenterWorld(cell); 

// GetCellCenterWorld实际是如下方式计算的，原先Tile Anchor的值位于offset
var cellcenter = Tilemap.LocalToWorld(Tilemap.CellToLocalInterpolated((Vector3)cell) + Tilemap.CellToLocalInterpolated(offset));   

//cellcenter这个值计算的xy是正确的，但z会出现问题，与Tilemap.transform.position.z不一致，需要手动赋值一遍
cellcenter.z  = Tilemap.transform.position.z;
