在Unity中可以通过如下方式加载场景

// 同步加载
public SceneInstance LoadScene(string path, LoadSceneMode mode)  
{  
    return Addressables.LoadSceneAsync(path, mode).WaitForCompletion();  
}

// 异步加载
public async UniTask<SceneInstance> LoadSceneAsync(string path, LoadSceneMode mode)  
{  
    return await Addressables.LoadSceneAsync(path, mode);  
}

但这两种方式均存在问题，如下

  1. 异步加同步加载时Update报错
// 这种情况下会出现如下报错
// Reentering the Update method is not allowed.  This can happen when calling WaitForCompletion on an operation while inside of a callback.
// 参考如下文件中的update的实现，Unity是为了阻止无限递归才禁用的  	//PackageCache\com.unity.addressables@1.21.9\Runtime\ResourceManager\ResourceManager.cs

var scene = await LoadSceneAsync("Assets/GameRes/Scenes/Menu.unity", LoadSceneMode.Single);
Addressables.LoadAssetAsync<GameObject>("Assets/GameRes/a.prefab").WaitForCompletion(); 

var ret = await Addressables.LoadSceneAsync(path, mode).ToUniTask(progress, PlayerLoopTiming.LastUpdate);  
await ret.ActivateAsync();
Addressables.LoadAssetAsync<GameObject>("Assets/GameRes/a.prefab").WaitForCompletion();
  1. 场景的Active存在延迟
// 此时UIRoot无法在Menu场景中生成，因为ActivateAsync这个过程还没结束
var scene = LoadScene("Assets/GameRes/Scenes/Menu.unity", LoadSceneMode.Single);  
var obj = new GameObject("UIRoot");

// 同上也是因为ActivateAsync这个过程还没结束
var scene = await LoadSceneAsync("Assets/GameRes/Scenes/Menu.unity", LoadSceneMode.Single);
var obj = new GameObject("UIRoot");

对于这两个问题其实都是由于Unity资源加载异步实现方式导致的，在Unity中资源加载完成后的回调是通过在Update中轮询完成的，即资源即便已经加载好了，这个时间也需要到下一次的Update中去派发事件。

所以可以采用UniTask中的await UniTask.Yield(PlayerLoopTiming.LastUpdate);在资源加载结束后使用该函数将任务移到Update的结尾处，这样便能确保ActivateAsync已经结束了，而且也可以跳出Update的报错。

// 异步加载
public async UniTask<SceneInstance> LoadSceneAsync(string path, LoadSceneMode mode)  
{  
    var ret = await Addressables.LoadSceneAsync(path, mode);  
    await ret.ActivateAsync();  // 确保加载完成
    await UniTask.Yield(PlayerLoopTiming.LastUpdate); // 跳到Update尾部，防止同步加载报错
    return ret;
}

显示进度

public async UniTask<SceneInstance> LoadSceneAsync(string path, LoadSceneMode mode, IProgress<float> progress)  
{  
    // 在ToUniTask中所指定的PlayerLoopTiming只影响progress的调用频率
    // 结束的触发则依赖于Addressables.LoadSceneAsync(path, mode)本身的结束事件，即Update轮询
    // 所以下面的三个Debug.Log(Time.frameCount);都在同一帧内执行
    var ret = await Addressables.LoadSceneAsync(path, mode).ToUniTask(progress, PlayerLoopTiming.LastUpdate);  
    Debug.Log(Time.frameCount);  
    await ret.ActivateAsync();  
    Debug.Log(Time.frameCount);  
    await UniTask.Yield(PlayerLoopTiming.LastUpdate);  
    Debug.Log(Time.frameCount);  
    return ret;  
}

注意点

场景切换卡死

切换场景时场景中存在同步加载的资源时导致卡死，原因如下

虽然文档说是因为 Scene 切换时不能同步是因为切换时会 UnloadUnusedAssets，但经过实验发现调用 Resources.UnloadUnusedAssets() 后立马去同步 Load 资源也不会卡死，应该是 Scene 切换时还有其他的操作导致 Lock 住了其他的资源加载。可以参考 Addressables AssetLoading is blocked by Async Scene Loading
卡死问题复现的代码

// 挂在在场景中的脚本，同步加载资源,假设该Scene为Level1
public void Awake()
{
	Addressables.LoadAssetAsync<GameObject>("Assets/GameRes/a.prefab").WaitForCompletion(); 
}

// 以下为场景切换代码
public void ChangeScene()
{
	// 进入Loading场景
    await LoadSceneAsync("Assets/Scenes/Loading.unity", LoadSceneMode.Single);  
    // 切换进Level1，此时会触发上面的Awake中的同步加载，导致死锁
	await LoadSceneAsync("Assets/GameRes/Scenes/Level1.unity",LoadSceneMode.Single);
}

解决方式是，若要使用同步加载需要预加载资源

public async void GameInit()
{
	// 预加载资源，由于Addressable中将缓存存于字典中，所以类型需要与一致才能在同步中直接取出
	Addressables.LoadAssetAsync<GameObject>("Assets/GameRes/a.prefab",false); 
}

还遇到过在切换场景后的 1-2 帧内 LoadAsync 某个特定资源是 Editor 卡死且 CPU 占用率为 0，且无法断点定位。
后发现可以等待几帧在去加载资源。
通过在 Groups 面板中删除资源，重新拖回去之后重新 Build 资源彻底解决了部分该问题，但至于为什么出现这个情况未知。
最终解决方方法还是需要等到场景彻底加载完之后再去请求资源加载才不会卡死，即 await ret.ActivateAsync(); 和同步加载时的策略一样。
