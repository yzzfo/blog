环境

  1. ubuntu22.04
  2. sonarr 3.0.9.1549

起因

在获取动漫数据时发现部分动漫没有Alternate Titles，于是想自己添加合适的Alternate Titles用于搜索(当前sonarr只会使用全英文的Title去搜索，会跳过非英文的Title)，查看源码后发现Sonarr是使用thexem.info来获取scene mapping数据的，但thexem.info只能是注册用户才能进行修改于是找到了thexem-proxy这个项目来解决这个问题。

Docker-Compose

    thexem-proxy:
      image: ghcr.io/uilton-oliveira/thexem-proxy:latest
      container_name: thexem-proxy
      volumes:
        - ./thexem-proxy/config:/thexem/config
      environment:
    #   - NODE_TLS_REJECT_UNAUTHORIZED=0
        - API_URL=172.67.160.240
        - HEADER_HOST=thexem.info
      networks:
        - main
      ports:
        - 3005:3005
      restart: unless-stopped

xem-mapping.yml

exclude:
# TVDB-ID
- '72241'
include:
  '326732':
  - Ze Tian Ji: -1
  'Tvdb-ID':
  - Example All Seasons Title 1: -1
  - Example All Seasons Title 2: -1
  - Example Season 1 Title: 1
  - Example Season 2 Title: 2
  - Example Season 3 Title: 3

配置DNSMASQ

address=/thexem.de/10.0.0.115

Nginx

server {
  listen        443;
  server_name   thexem.info;

  location / {
	proxy_pass http://thexem-proxy:3005/;
	proxy_set_header X-Real-IP $remote_addr;
	proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
	proxy_set_header X-Forwarded-Proto $scheme;
	proxy_set_header X-Forwarded-Host $http_host;
	proxy_redirect off;
  }
}

申请自签证书

mkdir ssl
cd ssl
nano localhost.conf

[req]
default_bits       = 2048
default_keyfile    = localhost.key
distinguished_name = req_distinguished_name
req_extensions     = req_ext
x509_extensions    = v3_ca

[req_distinguished_name]
countryName                 = Country Name (2 letter code)
countryName_default         = US
stateOrProvinceName         = State or Province Name (full name)
stateOrProvinceName_default = New York
localityName                = Locality Name (eg, city)
localityName_default        = Rochester
organizationName            = Organization Name (eg, company)
organizationName_default    = localhost
organizationalUnitName      = organizationalunit
organizationalUnitName_default = Development
commonName                  = Common Name (e.g. server FQDN or YOUR name)
commonName_default          = localhost
commonName_max              = 64

[req_ext]
subjectAltName = @alt_names

[v3_ca]
subjectAltName = @alt_names

[alt_names]
DNS.1   = thexem.info
DNS.2   = 192.168.3.2 #自己的ip

# 生成私钥key和证书crt
openssl req -x509 -nodes -days 365 -newkey rsa:2048 -keyout localhost.key -out localhost.crt -config localhost.conf

cp localhost.crt /etc/ssl/certs/localhost.crt

cp localhost.key /etc/ssl/private/localhost.key

# 将证书写入ca-certificates中，使curl之类的可以信任该https连接
cat /etc/ssl/certs/localhost.crt >> /etc/ssl/certs/ca-certificates.crt

配置Sonarr信任证书

到sonarr中执行如下命令即可

cert-sync /etc/ssl/certs/ca-certificates.crt
