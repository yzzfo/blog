保存凭证在本地

touch .git-credentials
echo ".git-credentials" >> .gitignore
git config --local credential.helper 'store --file .git-credentials'

查看配置

git config --global --list  #打印出global配置信息
git config --local --list  #打印出global配置信息

global 配置文件位置在 ~/.gitconfig ,默认凭证保存在 ~/.git-credentials

local 配置文件在 ./.git/config

Linux 下不同用户权限问题

# 使该用户可以编辑任何文件夹
git config --global --add safe.directory "*"


fatal: unable to get credential storage lock in 1000 ms: Permission denied

该问题可能是当前文件夹没给当前用户写的权限，例如如下结构，原因在于修改文件夹内的文件名/删/创都需要该文件夹的权限
resp (root:root) rw-r--r
  - .git-credential (www:www) rwxrwxrwx
  - others

fork其他项目

git clone <上游仓库地址>
# 设置远程仓库为<你的新仓库地址>
git remote set-url origin <你的新仓库地址>
git remote add upstream <上游仓库地址>

# 修改仓库内容  
#...
# git add --all
# git commit -m "..."

# 推送到origin main分支
git push origin main
#  推送，且设置默认远程分支为origin main
git push -u origin main

# 合并上游内容到当前仓库
git fetch upstream 

# 查看远程信息
git remote -v
# 查看当前分支的上游设置
git branch -vv
