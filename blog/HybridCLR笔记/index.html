遇到的问题

  1. Build 时出现 Link 相关的报错，例如 Link Hotfix.dll failed ，可能原因是在非热更程序集中用到了 Hotfix 中的类，例如在 Assembly-CSharp 中使用了 Hotfix 中的类型。

BuildTool

使用 BuildTool 时出现找不到 ManagedStripped 文件夹或者 miss script 等错误，原因在于 HybridCLR 源码在判断平台时采用的是 宏 进而导致无法在 A 平台下执行 B 平台构建所需要的事情，以及获得相关数据。关于在 Unity 中 miss script 本质的原因是在 Assembly 列表文件中，如 PC 端下的 ScriptingAssemblies.json 文件，而在 HybridCLR 中已经做了补全的操作，只不过因为宏被禁用了而已，所以解决方法很简单，只需要将代码中的宏改为通过 BuildTarget 枚举进行判断即可。例如 PatchScriptingAssemblyList.cs 文件

        public void OnPostprocessBuild(BuildReport report)  
        {            // 如果target为Android,由于已经在OnPostGenerateGradelAndroidProject中处理过，  
            // 这里不再重复处理  
// #if !UNITY_ANDROID && !UNITY_WEBGL  
            if (report.summary.platform != BuildTarget.Android && report.summary.platform != BuildTarget.WebGL)  
            {                PathScriptingAssembilesFile(report.summary.outputPath);  
            }// #endif  
        }
