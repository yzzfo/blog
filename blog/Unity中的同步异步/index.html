最近看接触xlua时发现一个挺有意思的例子,该例子中将原本的异步逻辑变为了同步逻辑,xlua中的实现是基于lua协程，于是我也想试一试在C#中实现类似的功能，lua协程简单来看也就是改变了调度顺序而已，看起来就像将异步方法放yeild中回调后续内容一样，在C#中类似的概念就是async和await，Unity在2017后就支持了C#中的async和await关键字，于是开始尝试着使用这两个关键字去实现这种功能。

async和await

使用规则

  1. async

    // async函数中必须包含await的方法
public async Task AsyncFunc()
{
      await xxx;
}

public void Call()
{
      await AsyncFunc();      //阻塞执行
      AsyncFunc();            //非阻塞       
}

  2. await
    await属于编译范畴的内容，可以被await的对象要符合一下条件

    • 存在一个GetAwaiter方法，该方法可以是对象自身的也可以是拓展方法，返回一个Awaiter对象
      • Awaiter对象也是一个编译范畴的概念，也要符合如下条件
        • 继承INotifyCompletion接口
        • 存在一个GetResult()方法，返回类型不限
        • 存在一个IsCompleted的bool变量或属性

    一个基本的例子

      public abstract class BaseAsync<T> : INotifyCompletion where T : BaseAsync<T>
  {
        public virtual bool IsCompleted => false;

        public abstract void OnCompleted(Action continuation);

        public T GetAwaiter()
        {
              return this as T;
        }
  }

  public abstract class VoidBaseAsync<T> : BaseAsync<VoidBaseAsync<T>>
  {
        public void GetResult()
        {
        }
  }

await的调用逻辑

先从await的调用逻辑看起

public class AsyncJob : INotifyCompletion
{
      public bool IsCompleted => false;

      public void OnCompleted(Action continuation)
      {
            continuation();
      }

      public AsyncJob GetAwaiter()
      {
            return this;
      }

      public void GetResult()
      {
      }
}

//编辑器中的代码
public async void Start()
{
      await new AsyncJob();
      Debug.Log("Async job");
}

//实际操作逻辑
public async void Start()
{
      var waiter = new AsyncJob().GetAwaiter();
      //IsCompleted就是用来判断那些还没开始就已经结束的任务
      if(waiter.IsCompleted)
      {
            waiter.GetResult();
      }else
      {
            //本质就是把下文内容给封成了委托用来注册回调
            waiter.OnCompleted(()=>{
                 //GetResult()等等
                 Debug.Log("Async job");
            });
      }
}

C#中实现xlua的asyncTosync

通过await的调用逻辑就不难发现，他其实和lua协程基本一样，await也不过是个委托封装器，专门把后续代码封装成委托的一个关键词罢了，那么思路就变成了通过await将后续方法注册给对应要等待的回调
伪代码

Button okBtn;
public void Confirm()
{
      await okBtn.OnClick;
      Debug.Log("Click");
}

实现线程同步

Unity中经常要进行线程同步操作，比如常用的事件模型，若想在网络回调中使用则需要在其中加入线程同步操作，通常来说都是维护一个任务队列之类的东西来进行同步，但通过awiat可以很容易实现线程切换。
await线程切换本质也就是把下文委托传递给不同线程去调用。

  1. 主线程
    public static SynchronizationContext UnityThreadContex { get; private set; }
public void Awake()
{
      UnityThreadContex = SynchronizationContext.Current;
}

public class MainJob : INotifyCompletion
{
      public bool IsCompleted => false;

      public void OnCompleted(Action continuation)
      {
            UnityThreadContex.Post((x) => continuation(), null);
      }

      public MainJob GetAwaiter()
      {
            return this;
      }

      public void GetResult()
      {
      }
}
  2. 线程池
    线程池思路也和上变一样，就是改变OnCompleted中的调用线程而已。

C#中Task的一些东西

Task是C#中自带的一些Await实现，他们大都执行时在调用时的SynchronizationContext.Current中，也有一些比如await Task.Delay(10)改变线程，究其原因在于Delay是采用另一线程来执行计数的，如果不是另一个线程那么也就阻塞本线程其他的东西了，那么async也就没什么意义了。
