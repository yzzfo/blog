Selenium的使用

Selenium虽然以前就接触过，但从未在Liunx服务器上去使用过，故在此记录一下在Linux下踩到的坑。

安装

既然是服务器上运行，那就还是使用Docker来安装比较方便了。Docker中提供了多种镜像，这里选择standalone-chrome就好，开箱即用，更多说明参考docker-selenium。
详细的Docker安装就不做赘述，这里使用Drone CI来使用Selenium，所以阐述下在Drone CI 2.0中的使用。

steps:
- name: test
  image: golang
  commands:
  - go run ./ -u http://chrome:4444/wd/hub  #selenium默认端口为4444

services:
- name: chrome
  image: selenium/standalone-chrome
  environment:
    DISPLAY: :1
  volumes:
  - name: shm
    path: /dev/shm
# 挂载shm用于selenium，推荐2G
volumes:
- name: shm
  host:
    path: /dev/shm

使用

在Golang中使用github.com/tebeka/selenium这个库操作Selenium，使用Chrome版的Selenium进行如下操作

	caps := selenium.Capabilities{
		"browserName": "chrome",
	}
    //在ssh等无图像输出的情况下，必须进行如下设置
    //不设置将导致各种问题
    //如DevTools not fount
    //source page crash等等
	chromeCaps := chrome.Capabilities{
		Path: "",
		Args: []string{
			"--headless", // <<<
			"--no-sandbox", //可以不特权模式执行docker
			"--user-agent=Mozilla/5.0 (Macintosh; Intel Mac OS X 10_13_2) AppleWebKit/604.4.7 (KHTML, like Gecko) Version/11.0.2 Safari/604.4.7",
			"--disable-gpu",
			"--disable-dev-shm-usage",  //设置后可以不挂载shm
		},
	}
	caps.AddChrome(chromeCaps)
	wd, err := selenium.NewRemote(caps, "http://chrome:4444/wd/hub")

注意事项

Selenium中输出的所谓只有本地链接才能使用WebDriver等报错一样的日志，仅仅是提示作用，在Docker中执行并没任何问题，无需理会。
