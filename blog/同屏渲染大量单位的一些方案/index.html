同屏渲染大量单位的一些方案.
Posted on 2023-02-18 :: Tags: Unity, 渲染

  最近遇到需要同屏渲染几百个单位的需求，在Unity中SkinnedMeshRender默认也没有合批这一说，首先就得解决一下这几百个Batch的问题。

NVIDIA 在 GDC 曾提出，25K batchs/sec 会吃满 1GHz 的 CPU，100%的使用率。

即有如下的估算公式

DrawCall_Num = 25K * CPU_Frame * CPU_Percentage / FPS
DrawCall_Num ： DrawCall数量（最大支持）
CPU_Frame : CPU 工作频率（GHZ单位）
CPU_Percentage：CPU 分配在drawcall这件事情上的时间率 （百分比）

例如：
2Ghz CPU，目标10%占用率，60帧
25000*2*10%/60 = 83.33
则drawcall为83个

Animation-Instance

Animation-Instance是Unity官方的一个插件，在对SkinnedMeshRender进行合批的同时也提供了其他常用的API，接入项目时比较方便。

AnimMap Baker

AnimMap Baker相较与Animation-Instance做法更加激进，直接将动画烘焙进了图片之中，所有动画均用顶点动画实现，压力全放于GPU上了，且API提供的较少接入项目较为麻烦。
