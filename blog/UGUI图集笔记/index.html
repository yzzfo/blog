  之前开发UI时都使用的NGUI或FairyGUI，最近在使用UGUI时发现打完包后图集资源始终无法加载出来，在一番调试之后找到了问题所在，所以来记录一下。

UGUI图集与NGUI的不同之处

NGUI

制作

NGUI打图集时需要用户指定所需的图片之后由程序生成一张大图作为图集

使用

NGUI使用图片时需要先传入图集之后选择里面的图片

UGUI

制作

UGUI制作图集时需要先创建Sprites Atlas文件，之后将需要的图片拖入其中即可，同一个Sprite 可以存在于多个图集之中

使用

直接将Sprite载入组件中即可

问题所在

从制作方式和使用方式中可以看出来，NGUI中资源是强关联的关系，在资源打包时是可以知道该预制体依赖哪个图集，UGUI中依赖Sprite，但Sprite是被图集依赖的对象，且可以存在于多个图集之中，这时就不能通过依赖分析的方式正确的加载图集文件了。

解决方法

指定依赖

在UI基类中指定一个字段填写所依赖的图集资源，在加载资源时同步加载所需图集。

使用SpriteAtlasManager

当一个Sprite打包进了图集但在运行期间无法找到该图集时触发，如果Sprite在多个图集中则随机选一个图集，该方法的坏处在于异步加载时会导致图片空几帧。

private void OnEnable()
{
    SpriteAtlasManager.atlasRequested += RequestAtlas;
}
​
private void OnDisable()
{
    SpriteAtlasManager.atlasRequested -= RequestAtlas;
}
​
private void RequestAtlas(string atlasName, Action<SpriteAtlas> callback)
{
    // 加载图集
    var sa = Resources.Load<SpriteAtlas>(atlasName);
    // 加载完成后执行callback通知界面刷新
    callback(sa);
}
