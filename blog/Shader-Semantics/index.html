  在学习Shader是经常会看到诸如TEXCOORD0,POSITION,这样的东西，这些东西叫做Semantics中文叫做语义。

作用

在Unity的shader中，TEXCOORD0，POSITION等都是用来表示顶点和片元的数据的语义。语义是用来指示HLSL shader程序中输入和输出变量的意图的。

例如，POSITION语义表示顶点在裁剪空间中的位置，以便GPU知道屏幕上的光栅化位置和深度。 TEXCOORD0，TEXCOORD1等语义用于表示任意高精度数据，如纹理坐标和位置。 顶点输出和片元输入的COLOR0和COLOR1语义用于表示低精度0到1范围的数据（如简单的颜色值）。

语义的规则或限制

  • 语义必须是从以下列表中选择的：POSITION，NORMAL，TANGENT，TEXCOORD0-7，COLOR0-1，SV_Target等。
  • 语义必须与变量的类型和维度匹配。例如，POSITION必须是float3或float4，COLOR0必须是float[2|3|4]等。
  • 语义不能重复使用。例如，不能有两个变量都使用TEXCOORD0语义。
  • 片元shader的输出语义通常是SV_Target，表示输出的颜色值。

可以参考MSDN上的Semantics文档来了解更多细节。

为什么要使用语义

使用这些语义的原因是，它们可以传递关于参数的预期用途的信息，从而使shader程序与GPU和其他shader阶段之间的通信更加清晰和有效。

例如，POSITION语义告诉GPU这个变量是用来表示顶点位置的，而TEXCOORD0语义告诉GPU这个变量是用来表示纹理坐标的。这样，GPU就可以根据语义来正确地处理和插值数据，并将其传递给下一个shader阶段（如片元shader）。

可以把语义看作是一种约定或标签，它们帮助组织和管理shader代码，并避免一些错误或混乱。

系统语义

为什么MSDN上TEXCOORD0为float4，使用时却可以声明为float2，但在规则或限制中却说了类型必须匹配？

之前说的POSITION必须是float3或float4类型，是指在顶点shader输出或片元shader输入时，使用的SV_POSITION语义，这个语义是用来表示顶点在裁剪空间或屏幕空间的位置的，它是GPU栅格化和深度测试所必需的

由于裁剪空间和屏幕空间都是三维或四维的，所以SV_POSITION语义必须使用float3或float4类型来声明。 如果使用其他类型，例如float2，那么GPU将无法正确地计算顶点的位置和深度，从而导致渲染错误。

COLOR也同理，指的shader片元输出时的SV_Target语义。

根据MSDN中所述DX10的SV_Target等效DX9的COLOR,所以在Unity中会看到一些宏定义将SV_Target定为COLOR

而TEXCOORD0语义则不同，它只是用来表示纹理坐标或其他任意数据的，并不影响GPU栅格化和深度测试。 因此，TEXCOORD0语义可以使用任何合适的类型来声明，只要与其他shader阶段之间的数据传递保持一致即可。

他们之间的差别就在于SV_POSITION语义是一个特殊的系统值语义（System-Value Semantics），它有一些固定的规则和约束，而TEXCOORD0语义则是一个普通的用户定义语义（User-Defined Semantics），它有更多的灵活性和自由度。

Note

SV_POSITION

以SV_POSITION为例，他在Vertex和Frag阶段时所代表的数据是不同的

struct v2f  
{  
    float4 vertex: SV_POSITION;  
};  
  
v2f vert(appdata v)  
{  
    v2f o;  
    o.vertex = UnityObjectToClipPos(v.vertex);  // 此时SV_POSITION为裁剪坐标系下的位置
    return o;  
}

fixed4 frag(v2f i): SV_Target  
{  
	// 此时SV_POSITION 为屏幕空间位置
	// 这个时候 i.vertex.xy 并不是裁剪坐标下的位置，而是屏幕位置
    return fixed4(i.vertex.xy/_ScreenParams.xy,0,1); //计算屏幕UV
}
